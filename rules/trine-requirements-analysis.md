# Requirements Analysis Rules (Phase 1.5)

## 핵심 원칙

기획서의 모호한 요구사항을 **구현 전에** 인터랙티브 Q&A로 해소한다.

## Q&A 프로세스

1. 기획서 초기 분석 → 불명확점 식별
2. 질문 과다 판정 (0/1~3/4~5/6+)
3. Q&A 실행 (번호 매기고 한 번에 제시)
4. 매트릭스 생성

## 질문 기준

**질문 대상**: 결과물에 영향을 미치는 불명확점만
- 수용 기준 없거나 모호한 FR
- 상충하는 요구사항
- 우선순위 미명시 핵심 기능
- 측정 기준 없는 NFR

**질문 비대상**:
- 기술적 구현 방식 (AI 자체 판단)
- 이미 명확한 요구사항 확인
- 기획서 범위 밖 제안

## 질문 수 판정

| 질문 수 | 판정 | 동작 |
|:-------:|------|------|
| 0개 | 명확 | Q&A 스킵 → 바로 매트릭스 |
| 1~3개 | 정상 | Q&A → 매트릭스 |
| 4~5개 | 경고 | Q&A + 기획서 보완 권고 |
| 6개+ | 반려 | [STOP] 기획서 반려 |

## 매트릭스 생성

- 프로젝트별 추출 스크립트 실행 (예: `node scripts/extract-requirements.mjs <doc>`)
- 출력: `.specify/traceability/{spec-name}-matrix.json`
- 파싱 실패 시 → AI 추출 모드 + [STOP] Human 확인

## 도메인 완결성 체크리스트

Q&A는 "모호함 제거"만 수행하므로, **구현 누락 예방**을 위해 아래 6개 영역의 완결성을 추가로 점검한다.
기획서/요구사항 분석 완료 후, 매트릭스 생성 전에 실행한다.

### 점검 대상

| # | 영역 | 점검 질문 | 미충족 시 |
|:-:|------|---------|----------|
| 1 | **CRUD 완결성** | 기획서에 명시된 모든 리소스(Entity)에 대해 Create, Read(목록+상세), Update, Delete가 정의되어 있는가? | 누락된 연산을 Q&A에 추가하거나, 의도적 제외 시 Spec에 "미지원" 명시 |
| 2 | **권한/인증** | 각 연산(API)에 인증 필요 여부와 역할별 접근 권한(RBAC)이 정의되어 있는가? | 누락된 권한 정의를 Q&A에 추가 |
| 3 | **에러 시나리오** | 주요 연산에 대해 예상 에러 상황(4xx/5xx)과 사용자 복구 경로가 정의되어 있는가? | 에러 시나리오가 없으면 Spec 섹션 4 Error Path에서 누락 발생 |
| 4 | **프론트엔드 상태** | 데이터를 표시하는 모든 화면에 로딩/에러/빈 상태(empty state) UI가 정의되어 있는가? | Spec 섹션 9.5에서 누락 발생 — 화면 목록에서 3-State 대상 식별 |
| 5 | **테스트 시나리오** | 핵심 비즈니스 로직에 대해 최소 정상 케이스 + 에러 케이스 테스트가 정의 가능한 수준으로 요구사항이 구체적인가? | Spec 섹션 10에서 시나리오 도출 불가 — 수용 기준(AC) 보강 필요 |
| 6 | **입력 검증 규칙** | 사용자 입력이 있는 모든 필드에 타입, 길이 제한, 형식, 허용 범위가 정의되어 있는가? | 프론트엔드+백엔드 양쪽의 검증 규칙 누락 — DTO 데코레이터 도출 불가 |

### 실행 규칙

1. Q&A 완료 후, 매트릭스 생성 **전**에 위 6개 영역을 점검한다
2. 미충족 항목이 있으면 **추가 질문을 생성**하거나, 의도적 미지원을 명시하도록 요청한다
3. 점검 결과는 매트릭스에 `completenessCheck` 필드로 기록한다
4. 6개 영역 모두 충족 시 매트릭스 생성으로 진행한다
5. Hotfix 분류는 해당 영역만 점검 (전체 6개 불필요)

### 분류별 적용 범위

| 분류 | 적용 영역 |
|------|---------|
| **Hotfix** | 해당 기능 영역의 에러 시나리오(#3)만 점검 |
| **Standard** | 6개 영역 전체 (Agent Teams로 병렬 처리 가능) |

## 비표준 기획서 Fallback

1. 추출 스크립트 파싱 실패
2. AI가 FR/NFR 추출 → 표준 형식 중간 파일 생성
3. [STOP] Human 확인 필수
4. 확인 후 재실행
5. session-state에 `requirementSource: "ai-extracted"` 기록
