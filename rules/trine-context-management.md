# Context Window Management Rules

## 핵심 원칙

200K 토큰 컨텍스트 중 **70% (약 140K)**를 실질적 상한선으로 관리한다.
성능 저하는 마지막 20%에서 비선형 가속되므로, 70% 초과 전에 대응한다.

## /compact 트리거 시점

| 시점 | 조건 | 행동 |
|------|------|------|
| Phase 2 → 3 전환 | Spec/Plan 작성 완료 후 구현 시작 전 | `/compact` 실행 권장 |
| AI Check 3.5 반복 | 자동 수정 루프 **3회 초과** | `/compact` 후 재시작 |
| 대량 파일 탐색 후 | Grep/Read 결과가 누적되었을 때 | `/compact` 실행 권장 |
| 체감 성능 저하 | 응답 품질 하락 감지 시 | `/compact` 즉시 실행 |

## 컨텍스트 절약 전략

### 1. 서브에이전트 위임 우선

대용량 코드베이스 탐색, 멀티파일 검색은 **Explore 서브에이전트에 위임**한다.
서브에이전트는 독립 컨텍스트를 사용하므로 메인 세션 컨텍스트를 소비하지 않는다.

### 2. 파일 읽기 최소화

- 전체 파일 대신 `offset`/`limit`으로 필요 부분만 읽기
- 이미 읽은 파일 내용을 반복 요청하지 않기
- 대형 로그/빌드 출력은 `head_limit`으로 제한

### 3. MCP 도구 토큰 인지

MCP 서버 상시 로드 시 도구 정의만으로 상당한 토큰 소비.
실제 작업 가용 컨텍스트는 **200K - MCP(도구 정의) - Rules** 기준으로 계산.

### 4. MCP Tool Search 활용

- `/mcp` 명령으로 미사용 MCP 서버 비활성화 → 도구 정의 토큰 절약
- MCP Tool Search (레이지 로딩)가 활성화되면 필요 시점까지 도구 로딩 지연
- `/context` 명령으로 토큰 사용량 상세 브레이크다운 확인

## 세션 관리 규칙

| 명령 | 용도 | 사용 시점 |
|------|------|----------|
| `/rename {feature-name}` | 세션에 설명적 이름 부여 | 작업 시작 시 (필수 습관) |
| `--continue` | 가장 최근 대화 이어하기 | 세션 재개 시 |
| `--resume` | 대화형 세션 선택기 | 특정 과거 세션 재개 시 |

### 세션 간 영속화

- 세션 종료 전 **git commit으로 작업 영속화** (세션 간 상태 보존)
- `/compact` 후에도 핵심 결정사항은 MEMORY.md에 기록
- 장기 분석은 서브에이전트 `resume` (agentId)으로 이어서 실행 가능

## AI 에이전트 행동 규칙

1. Phase 전환 시 `/compact` 실행 여부를 판단한다
2. AI Check 3.5 자동 수정 3회 초과 시 반드시 `/compact` 후 재시작
3. 3개 이상 파일 탐색 시 Explore 서브에이전트 위임을 우선 고려한다
4. 컨텍스트 부족 징후 (응답 누락, 반복 질문) 감지 시 즉시 `/compact`
5. 작업 시작 시 `/rename`으로 세션 이름 부여를 고려한다
6. `/context`로 토큰 사용 현황을 주기적으로 확인한다
